Storing Sensitive Information in URLs

This example demonstrates the problem of storing sensitive information, such as key material, in URLs.

The cryptographic key is kept using the hashlocation part of the URL. While this is not transmitted to the server, it is kept in the browser's history and also displayed in the URL bar. Furthermore, the user may not expect the URL to contain sensitive information, sharing it with others.

Do not use the URL to store or transmit sensitive information. Data that is supposed to be sent to the server can be sent in the body of a POST request. Data that is to be kept locally can be stored in local script variable or (using appropriate protections) in the localStorage.
