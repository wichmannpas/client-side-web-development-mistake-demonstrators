Insecure Input Fields

This example demonstrates the problem of using an insecure input field for sensitive values.

The password that is used for the key wrapping is asked using a regular text field rather than a password field. Thus, the value can usually be seen in plaintext by other users, and additionally may be stored for autocompletion by the browser.

This example uses a custom font consisting only of dots. Thus, the password is not readable in the text field. However, the value can still be copied and pasted into another text field, making it readable.

Always use a field of input type password for passwords or other sensitive information.
